页面目的：提供人因工程师的全局工作状态视图，一屏掌握所有项目进度与工作量。

布局结构
┌──────── 顶栏（角色标识 + 通知 + 切换角色）──────────────┐
├── 左侧导航 ─┬── 主内容区 ────────────────────────────────┤
│  看板       │                                            │
│  我的项目   │                                            │
│  被试库     │                                            │
│  数据归档   │                                            │
│  ──────    │                                            │
│  设置       │                                            │
└────────────┴────────────────────────────────────────────┘
主内容区：区块一 —— 个人工作量统计（顶行 4 卡）
指标卡	数据内容	可视化
本月进行中实验	项目数量	数字 + 与上月对比箭头
待处理任务	待完成事项数	数字 + 红点（如有紧急）
本月已完成实验	数量	数字 + 完成率环形图
总被试参与人次	跨所有项目累计	数字
主内容区：区块二 —— 项目进度看板（Kanban 视图）

每张项目卡显示：

主内容区：区块三 —— 近期任务时间线（右侧面板）

主内容区：区块四 —— 实验完成质量指标
折线图：横轴=时间（月），纵轴=以下3条线：

量表回收率
数据有效率（剔除异常值后）

H2 — 实验范式选择 {#h2}
URL：/tob/he/projects/new/paradigm
页面目的：根据研究类型，引导选择适配的实验范式与评测方法。

页面结构
左侧：已选研究类型列表（来自H1，可返回修改）
右侧：对应的可用实验范式列表

实验范式选项（根据研究类型动态展示）
握持舒适度 / 形态适配性 类型下：

范式	说明
主观量表评定	使用 Likert 量表/SD 语义差异量表，被试握持后评分
身体地图标注	被试在数字人体图上标注不适部位
比较判断法	多款样机两两对比，产出偏好排序
持续握持疲劳测试	指定时长握持后评估疲劳程度
热控制体验类型下：

范式	说明
温度主观感受量表	不同使用时长下的主观热感评分
热舒适区间评定	确定被试认为"舒适"的温度上下限
声学体验类型下：

范式	说明
音质主观评价	ITU-T P.800 等标准化主观听音实验
通话清晰度评定	特定语料下的主观清晰度评分
振动体验类型下：

范式	说明
振动强度感知量表	对不同马达参数的强度偏好与识别率
触觉纹理辨别	被试区分不同振动模式的准确率
每种范式卡片下方有「了解详情」展开说明（实验流程简述、适用样本量、预计时长）。

被试量估算（右下角实时计算）
根据所选范式和研究类型，实时显示推荐被试数量区间（如「建议 20–30 人」）。

H3-6 — 数据分析与结论 {#h3-6}
URL：/tob/he/projects/:id/run/analysis

页面目的：对采集完成的所有数据进行自动预处理、统计分析，并产出可直接使用的研究结论报告。

区块一：数据预处理（自动执行）
系统自动完成，结果用日志形式展示：

✅ 量表数据导入：20份，有效19份（剔除P015：作答时间极端异常）
✅ 录像数据匹配：19段视频已与被试ID关联
✅ 缺失值处理：2处空值已用均值填充，已标注
✅ 信度检验（Cronbach α）：0.87（量表内部一致性良好）
⚠️  P008 握持压力数据存在离群值，已自动标注，建议人工核查
区块二：统计分析结果（自动生成）
根据研究类型自动选择统计方法并输出：

描述统计

各维度得分均值、标准差、中位数
箱线图（可视化评分分布）
推断统计（视研究设计而定）

单样本 t 检验 / 配对 t 检验 / 单因素方差分析
显著性结果自动标注（p < 0.05 ✅ / p < 0.001 ✅✅）
